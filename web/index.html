The Problem

Why Discord Rewrote
Their Backend in Rust

🗑️

Go's garbage collector froze everything

Every ~2 minutes, Go paused all goroutines to scan memory. At Discord's scale, every user felt a simultaneous stutter. There's no tuning your way out — it's baked into Go.

🔒

One lock, millions of waiters

A single RwLock on a HashMap means every user's read or write queues behind everyone else. 5,000 users = 5,000 threads waiting in a single line.

💾

Millions of tiny database writes

Tracking which message each user last read creates millions of writes per second at scale. Naively writing each one instantly melts the database.

✅

RustRelay solves all three

No GC, DashMap sharding for lock-free concurrency, and a write-buffer that batches 50,000 DB writes into a handful of bulk inserts every 5 seconds.

Engineering

Seven Hard Problems,
Seven Rust Solutions

Each subsystem solves a real engineering challenge that Discord, Slack, and every large real-time platform had to tackle.

PROBLEM 01

Zero GC Pauses

Rust's ownership system frees memory the instant it's no longer needed — at compile time, not at runtime. Every struct, every temporary buffer is freed deterministically. No stop-the-world scan, ever.

Borrow checker
PROBLEM 02

Lock-Free Session Store

DashMap splits session data into 64 independent shards. Alice on shard #12 and Bob on shard #47 never block each other. Contention drops by 64× compared to a single RwLock.

gateway/session.rs
PROBLEM 03

Channel Member Cache

Fan-out requires knowing who's in each channel. A 5-minute TTL cache means the first message triggers a DB query — the next 1,000 hit an in-memory HashMap. Query rate drops by 99.9%.

router/fanout.rs
PROBLEM 04

Presence Debounce

Phone through a tunnel = 2-second dropout. Without debouncing, that fires two presence events — "offline" + "online" — to 50,000 followers. A 5-second timer silently absorbs brief disconnects.

presence/tracker.rs
PROBLEM 05

Batched Presence Fanout

If 10 users come online simultaneously and you share servers with each of them, that's O(n²) notifications. RustRelay collects all changes every 100ms, deduplicates, and fires one burst.

presence/broadcast.rs
PROBLEM 06

Read State Write Buffer

The exact service Discord rewrote. Every "mark as read" is buffered in memory (microseconds). Every 5 seconds, 50,000 individual acks become a handful of bulk UPSERT queries. The DB never feels the load.

readstate/cache.rs
PROBLEM 07

Heartbeat Reaper

Laptops close without disconnecting. Phones switch networks silently. A background task checks every 10 seconds — sessions silent for 60s are reaped instantly, memory freed in microseconds, not GC cycles.

gateway/heartbeat.rs
BONUS

Redis Cross-Node Fanout

Alice on node-1 sends a message. Bob is connected to node-2. RustRelay publishes to a Redis channel — node-2 picks it up and delivers to Bob. No direct node-to-node links needed.

router/redis_bridge.rs
